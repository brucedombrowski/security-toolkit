\dmsection{Certificate Handling}

\SoftwareName{} implements secure certificate filtering to ensure only appropriate signing certificates are used:

\begin{itemize}
    \item \textbf{Expiration Validation:} Expired certificates are rejected
    \item \textbf{Key Usage:} Digital Signature key usage flag required
    \item \textbf{Enhanced Key Usage (EKU):}
    \begin{itemize}
        \item Email Protection (OID 1.3.6.1.5.5.7.3.4)
        \item Document Signing (OID 1.3.6.1.4.1.311.10.3.12)
    \end{itemize}
    \item \textbf{Government Certificate Priority:} PIV/CAC certificates (DOD, NASA, FPKI) displayed first
    \item \textbf{Excluded Issuers:} VPN and device certificates filtered (Palo Alto, Cisco, Zscaler, etc.)
\end{itemize}

\dmsection{Security Controls}

The following security controls are implemented in \SoftwareName{}:

\begin{itemize}
    \item \textbf{No Private Key Logging:} Private key material is never logged or displayed
    \item \textbf{Secure PIN Entry:} PIN prompts handled by Windows secure dialogs
    \item \textbf{Read-Only Store Access:} Certificate store accessed in read-only mode
    \item \textbf{Sensitive File Exclusion:} gitignore excludes *.pem, *.p12, *.pfx, *.key files
    \item \textbf{Append-Mode Signing:} Existing signatures preserved in multi-signature workflows
    \item \textbf{No Network Communication:} Application operates entirely offline
\end{itemize}

\dmsection{U.S. Export Control Compliance}

\SoftwareName{} does not implement or distribute cryptographic functionality for data confidentiality (encryption). All cryptographic operations are limited to digital signature generation, verification, and hashing for integrity and authentication purposes only.

As such, \SoftwareName{} is \textbf{not subject to U.S. Export Administration Regulations (EAR) encryption controls} under 15 CFR 740.13(e). The software is not classified under ECCN 5D002 and no export licenses are required for distribution.
